# ROS 2

## 1. ROS 2 사용 이유
- ROS 1 개발 중단, ROS 2 비중 증가
- 특징
	- 시장 출시 시간 단축
	- 생산을 위한 설계
	- 멀티 플랫폼 (Linux, Windows, macOS)
	- 다중 도메인
	- 벤더 선택 가능
	- 공개 표준 기반
	- 오픈소스 라이센스 채택 (Apache 2.0)
	- 글로벌 커뮤니티
	- 산업 지원
	- ROS 1과의 상호 운용성 확보
- ROS 1 한계점 -> ROS 2 변경
	- 단일 로봇 -> 복수대의 로봇
	- 워크스테이션급 컴퓨터 -> 임베디드 시스템에서 사용
	- Linux 환경 -> 멀티 플랫폼
	- 실시간 제어 불가 -> 실시간 제어 가능
	- 안정된 네트워크 환경 요구 -> 불안정한 네트워크 동작 가능
